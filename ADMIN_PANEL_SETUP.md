# پنل مدیریت وب — راه‌اندازی

فایل `docs/index.html` همان پنل مدیریتی است که فرستادید — بدون تغییر. برای اینکه از داخل اپلیکیشن قابل بازکردن باشد، باید آن را جایی روی اینترنت میزبانی (host) کنید تا یک آدرس (URL) داشته باشد. ساده‌ترین راه، استفاده از **GitHub Pages** است، چون از قبل یک ریپازیتوری روی گیت‌هاب دارید.

## مرحله ۱: فعال‌سازی GitHub Pages

1. وارد ریپازیتوری اپلیکیشن‌تان در github.com شوید.
2. تب **Settings** (بالای صفحه ریپو) را باز کنید.
3. از منوی سمت چپ روی **Pages** کلیک کنید.
4. زیر «Build and deployment» → «Source»، گزینه **Deploy from a branch** را انتخاب کنید.
5. شاخه (Branch) را روی **main** بگذارید و پوشه را روی **/docs** تنظیم کنید (نه `/root`).
6. **Save** بزنید.
7. چند ثانیه تا یک دقیقه صبر کنید؛ صفحه را رفرش کنید تا آدرس سایت شما (چیزی شبیه `https://saeid4184.github.io/REPO-NAME/`) نمایش داده شود.

## مرحله ۲: وارد کردن آدرس در اپلیکیشن

1. اپلیکیشن را باز کنید → آیکون تنظیمات (⚙️) → بخش **«پنل مدیریت وب»**.
2. آدرسی که از GitHub Pages گرفتید را در کادر وارد کنید و **ثبت** را بزنید.
3. از حالا، آیکون 🌐 در بالای صفحه اصلی، این پنل را در مرورگر گوشی باز می‌کند.

## اطلاعات ورود پیش‌فرض پنل

طبق طراحی خود فایل:
- **نام کاربری:** `admin`
- **رمز عبور:** `admin1234`

⚠️ چون این نام‌کاربری/رمز مستقیم داخل کد جاوااسکریپت صفحه نوشته شده‌اند (نه یک سرور واقعی)، هرکسی که کد صفحه را ببیند می‌تواند آن‌ها را پیدا کند. برای یک ابزار داخلی/خصوصی این معمولاً کافی است، اما اگر می‌خواهید امنیت واقعی داشته باشید (مثلاً پنل روی اینترنت عمومی در دسترس همه باشد)، باید این بخش را با یک سیستم ورود واقعی (Firebase Auth یا مشابه) جایگزین کنید.

## نکته مهم درباره داده‌ها

این پنل، همان‌طور که طراحی شده، از **داده‌های نمایشی (fake/demo)** استفاده می‌کند — یک لیست ثابت در کد جاوااسکریپت به اسم `directoryData`، به‌همراه یک «شبیه‌ساز ترافیک» که هر ۴ ثانیه یک ورود/خروج ساختگی تولید می‌کند. **این پنل به دیتابیس واقعی گوشی‌های شما (Room/SQLite) وصل نیست** و آمار واقعی کارخانه را نشان نمی‌دهد؛ صرفاً یک نسخه‌ی نمایشی/دمو از طراحی است.

برای اینکه این پنل واقعاً داده‌های زنده‌ی کارخانه را نشان دهد، به یک زیرساخت مرکزی (مثلاً Firebase یا یک سرور اختصاصی) نیاز است که هم اپلیکیشن اندروید و هم این صفحه‌ی وب به آن متصل شوند و داده رد و بدل کنند. این یک پروژه‌ی جداگانه و بزرگ‌تر است — اگر به آن علاقه‌مندید، بگویید تا مرحله‌به‌مرحله برایتان طراحی‌اش کنم.

## کلید API هوش مصنوعی در این پنل

تب «دستیار هوشمند Gemini» داخل این پنل هم یک کادر مجزا برای کلید Gemini دارد (جدا از کلیدی که در اپلیکیشن اندروید وارد کرده‌اید). چون این صفحه در مرورگر اجرا می‌شود، هر کلیدی که آنجا وارد کنید در درخواست‌های شبکه قابل مشاهده است — پس یک کلید جداگانه و کم‌ریسک برای این پنل در نظر بگیرید، نه کلید حساس.
